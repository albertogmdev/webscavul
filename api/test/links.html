Prueba de Vulnerabilidades en Enlaces

Enlaces Inseguros (HTTP)

Estos enlaces deberían ser detectados como vulnerabilidades de "Link externo inseguro (HTTP)".

  • Enlace HTTP 1
  • Enlace HTTP 2
  • Enlace HTTP 3

Enlaces con target="_blank" sin rel="noopener noreferrer"

Estos enlaces deberían ser detectados por no tener los atributos 'rel' adecuados.

  • Blank sin rel
  • Blank con solo noopener
  • Blank con solo noreferrer
  • Blank con rel="follow"

Enlaces Seguros y Bien Configurados

Estos enlaces NO deberían generar ninguna vulnerabilidad.

  • Enlace HTTPS Seguro
  • Enlace Interno
  • Blank con noopener noreferrer
  • Blank con multiples rel

Otros Casos (sin href o href vacío)

Estos enlaces NO deberían generar la vulnerabilidad HTTP.

  • Enlace sin href (ancla)
  • Enlace con href vacío
